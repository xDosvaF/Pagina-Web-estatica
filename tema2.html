Introducción

Una API (Application Programming Interface) es un conjunto de reglas que permite que dos aplicaciones se comuniquen. Las APIs REST son el estándar más usado en la web por su simplicidad y compatibilidad con HTTP.

¿Qué es REST?

REST es un estilo arquitectónico para construir servicios web. Sus principios clave son:

  • Sin estado: cada petición contiene toda la información necesaria
  • Interfaz uniforme: uso de verbos HTTP estándar
  • Recursos identificables: cada recurso tiene una URL única

Verbos HTTP

VerboAcciónEjemplo

Autenticación

Proteger los endpoints de una API es fundamental. Los mecanismos más comunes son:

API Keys

JWT / OAuth

Buenas Prácticas

  1. Versionar la API (ej. /api/v1/)
  2. Usar códigos de estado HTTP correctos
  3. Devolver errores claros en formato JSON
  4. Implementar paginación en listas largas
  5. Documentar con Swagger u OpenAPI

Conclusión

Las APIs REST son el puente que conecta sistemas modernos. Un buen diseño mejora la seguridad, el mantenimiento y la experiencia de quien consume la API.

API Keys

  • Cadena secreta enviada en el header o URL
  • Fácil de implementar
  • Si se filtra, cualquiera puede usarla
  • Siempre usar con HTTPS

JWT / OAuth 2.0

  • JWT: token firmado con identidad del usuario
  • OAuth 2.0: autorización delegada (ej. "Entrar con Google")
  • Más seguros que las API Keys simples
  • Los tokens tienen fecha de expiración
